Architecture

什麼是
「前後端」？

Front-end 前端

餐廳的裝潢、菜單、服務生
使用者看得到、摸得到的部分 (HTML/CSS)

Back-end 後端

廚房、冰箱、廚師
處理資料、邏輯、儲存的地方

為什麼要分開？這樣你的 App 才能「記住」數據，或處理更複雜的計算。

GitHub

GitHub：
你的代碼保險箱
與展示櫃

  • 紀錄代碼的每一個版本（不怕改壞）。
  • 讓別人看到你的作品。
  • 最重要：它是通往雲端部署的鑰匙。
Deployment

認識 Antigravity：
讓部署像呼吸一樣簡單

  • 以前要讓網頁上線，需要設定伺服器、網域，非常痛苦。
  • Antigravity 是連接 GitHub 與雲端的「重力消除器」。
  • 只要把代碼推到 GitHub，它就會自動幫你把網頁「飛」上雲端。

結果：你會得到一個網址（例如 my-wbc-app.com），任何人都能打開。

Backend & DB

邁向更強大的應用：
後端與資料庫

  • 如果你的 App 需要「存檔」功能（例如紀錄最高分排行榜）：
  • 你需要一個資料庫 (Database)。

別擔心！在 2026 年，你只需要對 AI 說：
「幫我加上後端存檔功能，並連接資料庫」
AI 會幫你處理複雜的串接。
